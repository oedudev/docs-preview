#####   
**Fluxo pré Upluxo:**

São identificados os Tecidos que não serão mais usados no hospital

Tecidos são descontaminados na lavanderia

Facilities faz a triagem do que pode ser doado

O que pode ser doado é cedido para hospitais públicos

##### **Fluxo Upluxo (material)**:

Todo Resido Têxtil que não pode ser doado ou não tem demanda de doação entra no fluxo do UpLuxo

1. **Estoque do Hospital:**
    1. O Material nasce na plataforma como Estoque;
    2. Para adicionar um Material ao Estoque é necessário informar Peso, Tipo, Cor e Foto;
    3. O tipo do Material é selecionado entre uma relação disponível. (Implica a existência de um cadastro para tipos de materiais).
2. **Destinação**
    1. O Hospital combina a destinação de Material com um Upcycler com o qual tem Parceria
    2. No momento da entrega do material, Hospital seleciona Upcycler, Tipo, e Quantidade(KG) sendo retirado
    3. Uma MTR é emitida pelo hospital
    4. Automaticamente a quantidade de Material é baixada do Estoque do Hospital e adicionada ao Estoque do Upcycler

3. **Estoque do Upcycler**
    1. Cada Upcycler tem um “Sub Estoque”\* para cada Hospital com o qual tem Parceria
    2. Cada “Sub Estoque” registra tanto Materiais disponíveis para uso quanto “Restos de Produção” (Tecido não aproveitável para Produtos)
    3. Não há transferência de Material entre os “Sub Estoques”
    4. Cada Upcycler precisa dar baixa em seus "Restos de Produção", indicando como o destinou
    
    \*Hospital faz um Pedido, que é recebido pelo Upcycler\* -&gt; Fluxo paralelo
4. **Produção do Pedido**
    1. Upcycler sinaliza quem são os Colaboradores que fazem parte da “Equipe de Produção” e produzirão o Pedido
    2. Uma quantidade de Material é consumida na produção daquele Pedido
    3. Upcycler registra o peso total dos Produtos, dando baixa no “Sub Estoque”
    4. Upcycler compara o peso do Material Bruto com o peso dos Produtos
    5. Upcycler registra a diferença de peso no “Sub Estoque” como “Restos de Produção”
5. **Emissão de NF e Entrega do Pedido**
    1. Hospital informa número do pedido (Número de Pedido Comercial\*\*) para que Upcycler emita a NF
    2. Upcycler gera o QR CODE de validação da procedência
    3. Upcycler Anexa no sistema NF
    4. Upcycler Faz expedição do pedido e sinaliza no sistema
6. **Recebimento pelo Hospital Cliente**
    1. Hospital sinalizar que recebeu o pedido
    2. Hospital confirma a quantidade de brindes recebidos (para fins de Big Numbers)
7. **Tratamento de Restos de Produção**
    1. Upcycler realiza a destinação de uma quantidade de “Restos de Produção” para desfibramento
    2. Upcycler registra a destinação de “Restos de Produção”, dando baixa daquela Quantidade no “Sub Estoque”

\*Sempre que “Sub Estoque” é mencionado, entenda-se “Um compartimento de estoque que faz gestão de materiais oriundos de um único Hospital”

Ex: Um Upcycler pode ter um Sub Estoque apenas com material do Albert Einstein, outro apenas com material do Sírio-Libanês, etc...

\*\*Embora o sistema gere um ID interno para cada pedido realizado, “Número de Pedido Comercial” é um número separado e independente que identifica um dado pedido dentro do financeiro do Hospital e que deverá constar na NF. Esse número é apenas informado ao sistema, não é gerado por ele.

##### **Parcerias hospital – upcycler**

- O Hospital Cliente e o Upcycler precisam firmar uma parceria no sistema, sinalizando que podem trabalhar juntos;
- Para criar parceria:

1. Hospital encontra o Upcycler com o qual quer firmar Parceria
2. Hospital Envia “Solicitação de Parceria” para o Upcycler (similar à “solicitação de amizade” em Redes sociais)
3. Upcycler Aceita a solicitação de Parceria e as conferências e detalhes são resolvidas fora do UpLuxo
4. Hospital Confirma o vínculo de Parceria após todos os detalhes acertados
5. Hospital e Upcycler são registrados no sistema como Parceiros

##### **Rastreamento de procedência**

- O relatório de procedência deve:

1. 1. Identificar o Produto
    2. Identificar a Equipe de Produção
    3. Deve identificar o Upcycler
    4. Deve identificar o Hospital de origem do material

##### **Extras:**

- Hospitais podem cadastrar vários usuários. Cada hospital terá uma conta admin. Por sua vez, a conta admin poderá criar e gerir permissões de subcontas dentro do domínio do hospital;
- Os pedidos realizados pelo upcycler devem passar pelos status: Pendente, A Retirar Material, Em produção, Finalizado e Entregue;
- Deve haver a possibilidade de customizar os status;
- Cada pedido (lote) terá um ID no sistema Upluxo e um ID comercial, gerado pelo departamento de vendas do hospital;
- Upcycler deve poder fazer pedido de material a um Hospital, isso dispara apenas uma notificação para sinalizar aos Hospitais essa necessidade.
